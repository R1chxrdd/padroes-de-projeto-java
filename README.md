# Java Design Patterns

Projeto desenvolvido em **Java**, com foco na implementação prática, estudo e aplicação dos principais padrões de projeto arquiteturais descritos pelo GoF (Gang of Four).

## Sobre o Projeto

O repositório **Java Design Patterns** foi desenvolvido com o objetivo de servir como um catálogo prático para a resolução de problemas comuns no desenvolvimento de software. 

O projeto aplica conceitos clássicos de engenharia de software através de códigos simples e diretos, promovendo o desacoplamento, a reutilização de código e a manutenção facilitada.

## Padrões Implementados

- **Singleton** (Criacional)
- **Factory Method** (Criacional)
- **Abstract Factory** (Criacional)
- **Bridge** (Estrutural)
- **Decorator** (Estrutural)
- **Observer** (Comportamental)
- **Template Method** (Comportamental)

## Tecnologias Utilizadas

- Java (JDK 17 ou superior recomendado)
- IDE de sua preferência (IntelliJ, Eclipse, VS Code)

## Estrutura do Projeto

- **Pastas por Padrão** → Cada diretório (`/singleton`, `/observer`, etc.) contém um escopo isolado.
- **Interfaces / Abstrações** → Contratos que definem o comportamento base.
- **Implementações Concretas** → Classes reais que implementam as lógicas dos padrões.
- **Client / Main** → Classe responsável por rodar e demonstrar o funcionamento prático do padrão.

## Conceitos Aplicados

- Padrões de Projeto (Design Patterns)
- Programação Orientada a Objetos (POO)
- Princípios SOLID
- Composição vs Herança
- Baixo acoplamento e alta coesão
